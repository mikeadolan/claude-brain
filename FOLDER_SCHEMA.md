### Hook Chain

| Hook | Fires When | Reads | Writes |
|------|-----------|-------|--------|
| session-start.py | Session opens | NEXT_SESSION.md, DB (sessions, summaries) | additionalContext (checklist + notes + context) |
| user-prompt-submit.py | Before every user message | User prompt, DB (FTS5) | additionalContext (memories + GO check + frustration detector) |
| stop.py | After every Claude response | Session JSONL | DB via write_exchange.py |
| session-end.py | Session ends | — | brain_sync.py (detached backup) |

### MCP Server

| Component | Reads | Tools | Registered With |
|-----------|-------|-------|----------------|
| mcp/server.py | config.yaml, claude-brain.db | 11 read-only tools | ~/.claude.json (per-project) |

### Lifecycle Chains

**Session opens:**
`settings.json` → `session-start.py` → reads NEXT_SESSION.md + DB → injects additionalContext → Claude

**User sends prompt:**
`settings.json` → `user-prompt-submit.py` → DB FTS5 query + GO check → additionalContext → Claude

**Claude responds:**
`settings.json` → `stop.py` → `write_exchange.py` → DB writes + embeddings

**Session ends:**
`settings.json` → `session-end.py` → `brain_sync.py` (detached) → db-backup/

**End-session protocol (manual):**
write_session_notes.py → write_project_summary.py → MEMORY.md → governance files → NEXT_SESSION.md → git commit + push
